# Policy & Support RAG Assistant

A demo for Landmark Group (Gulf/India multi-brand retailer) showing a real
retrieval-augmented-generation (RAG) support assistant: it answers customer
questions about policy documents (returns, exchanges, warranties, etc.)
grounded strictly in the retrieved source text, with citations back to the
source document.

This is part of the Landmark e-commerce AI consulting demo suite
(`15-policy-rag-assistant`), meant to showcase real Python + RAG + REST API
capability rather than a mocked/fake response.

## What it demonstrates

- Chunking + embedding a small corpus of policy documents (`docs/`) with
  OpenAI embeddings.
- Semantic retrieval of the top-k most relevant chunks for a user question.
- A grounded answer generated by `gpt-4o`, constrained to only use the
  retrieved chunks (no hallucinated policy details), with citations.
- A REST API a frontend (or any client) can call directly.
- Full transparency for demo audiences: the API response includes the raw
  `retrieved_chunks`, not just the final answer, so you can show the
  retrieval step working, not just trust the output.

## Architecture

```
docs/*.md                 Source policy documents (plain markdown)
        |
        v
backend/ingest.py          Chunks docs, embeds each chunk via OpenAI,
                            writes an index (e.g. backend/index.json)
        |
        v
backend/rag.py              retrieve(query, k=3) -> top-k chunks by
                            embedding similarity
                             index_exists() -> bool
        |
        v
backend/main.py (FastAPI)    On startup: build the index if missing
                             POST /ask   -> retrieve + gpt-4o (JSON mode)
                                           -> grounded answer + citations
                                           -> retrieved_chunks (transparency)
                             GET  /health -> { status, index_ready }
        |
        v
Frontend (separate app)     Calls http://localhost:8010/ask
```

Design principles:

- **Grounded, not generic**: the system prompt instructs the model to answer
  ONLY from the retrieved context and to cite the source document by name.
  If the retrieved context doesn't cover the question, the model is
  instructed to say so rather than guess.
- **Fails loud in logs, fails clean to clients**: a missing `OPENAI_API_KEY`
  or any retrieval/model error never crashes the server or leaks a raw stack
  trace — every error path returns `{"error": "..."}"` with an appropriate
  HTTP status code. The server itself stays up so `/health` always answers.
- **Demo transparency**: `/ask` responses include `retrieved_chunks` so you
  can show an audience exactly what was retrieved before the model answered
  — this is the most convincing part of a RAG demo.

## Run instructions

```
cd 15-policy-rag-assistant
python -m venv venv
venv\Scripts\activate   (Windows)
pip install -r requirements.txt
set OPENAI_API_KEY=sk-...   (Windows)
uvicorn backend.main:app --reload --port 8010
```

On startup, the server checks whether an index already exists
(`rag.index_exists()`). If not, and `OPENAI_API_KEY` is set, it builds the
index automatically from `docs/` (`ingest.build_index()`). If the key is
missing, the server still starts (so you can hit `/health`), but `/ask` will
return a structured `503` error until the key is set and the server is
restarted.

The frontend (built separately) calls `http://localhost:8010/ask` directly —
CORS is wide open on this API since it's a local demo, not a production
deployment.

## API

### `GET /health`

```json
{ "status": "ok", "index_ready": true }
```

### `POST /ask`

Request:

```json
{
  "question": "Can I return a product I bought on sale?",
  "history": [
    { "role": "user", "content": "What is your return policy?" },
    { "role": "assistant", "content": "..." }
  ]
}
```

`history` is optional and used only to give the model conversational
context; grounding always comes from a fresh retrieval against `question`.

Success response (`200`):

```json
{
  "answer": "Sale items can be returned within 7 days of purchase, provided they are unused and in original packaging, according to the Returns & Exchange Policy...",
  "citations": [
    { "source": "returns-and-exchange-policy.md", "section": "Sale Items" }
  ],
  "retrieved_chunks": [
    {
      "text": "...",
      "source": "returns-and-exchange-policy.md",
      "section": "Sale Items",
      "score": 0.87
    }
  ]
}
```

Error responses (never a raw 500 stack trace) look like:

```json
{ "error": "OPENAI_API_KEY is not configured on the server. ..." }
```

with status codes such as `400` (bad request), `404` (nothing retrieved),
`502` (OpenAI/model failure), or `503` (index/key not ready).

## Files

- `backend/main.py` — FastAPI app, `/ask` and `/health` routes, startup index
  check, OpenAI call, error handling.
- `backend/rag.py` — `retrieve(query, k=3)` and `index_exists()`.
- `backend/ingest.py` — `build_index()`, reads `docs/`, embeds via OpenAI,
  writes the index file backend/rag.py reads from.
- `docs/` — source policy documents (markdown).
